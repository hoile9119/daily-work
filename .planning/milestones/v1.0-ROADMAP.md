## Milestone Summary

**Key Decisions:**
- Decision: Copilot CLI skill (not standalone Node.js) for sync — Notion MCP OAuth is session-managed; standalone scripts cannot authenticate
- Decision: Upsert keyed on (project + task name) — deterministic, no external ID needed
- Decision: AI summaries generated in batch before Notion writes — avoids context switching mid-sync
- Decision: `syncPrep.mjs` emits JSON to stdout, Claude handles Notion calls — clean separation of parse vs. sync
- Decision: Projects DB fetched once into in-memory name→ID map — O(1) API calls regardless of project count

**Issues Resolved:**
- Notion search returns workspace-wide results (not DB-scoped) — fixed by post-filtering on `parent.database_id`
- `in_trash` not supported via MCP tool — manual deletion required for Phase 2 test pages
- Relation format required UUID with dashes stripped: `33eb641fc7478196a41ff5b9a1801a2e`

**Technical Debt Incurred:**
- SYNC-02 (idempotent re-run / upsert on second run) not fully tested live — only CREATE path exercised; UPDATE path exists in SKILL.md but not live-validated
- SYNC-06 (2000-char chunking for Raw Notes) implemented in SKILL.md but not exercised with large notes
- 2 test pages from Phase 2 pending manual deletion in Notion UI
